Как запустить:
1) создать новую бд
2) заполнить в config.yaml data_base_settings addr, user, password, db_name
3) schema --config=./config.yaml - заполнит publuc схему в указанной бд и создаст двух тестовых юзеров с id 1 и 2
4) serve --config=./config.yaml - запускает сервер

Как работает:
Единстсвенный метод POST /api/transaction/run { "user_id": 1, "transaction_type": "0", "amount": 1000 } где transaction_type 0 - пополнение, 1 - списание.
Ответ: 400 и ошибка - неверное тело, 404 - не удалось найти пользователя, 402 - не достаточно средств, 200 и сумма нового баланса - транзакиця прошла.

Транзакиця сразу добавляется в бд. Если сервер упадет, то она останется там со статусом in_progress и при следующем включении сервер достанет и обработает 
все транзакции такого статуса (сначала на пололнение).
При поступлении запроса user берется из бд по id и ложится в мапу (кеш). В дальнейшем при поступлении запроса информация берется из этого кеша. 
(удалится из кеша через какое-то время в случае неактивности)
Для каждого пользователя свой мьютикс гарантирует очередность выполнения запросов.
Реализован Graceful Shutdown. Логгирование простое в консоль.
Уровневое логгирование в файлы и тесты не добавлены в связи с отсутствием времени.
